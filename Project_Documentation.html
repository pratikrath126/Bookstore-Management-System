1. Introduction

The Bookstore Management System is a full-stack web application that enables users to browse, search, and order books online, while providing administrators with a comprehensive dashboard for managing the bookstore inventory, categories, and orders. The application implements secure JWT-based authentication with role-based access control (USER and ADMIN roles).

1.1 Problem Statement

Managing a bookstore manually is time-consuming and error-prone. This application addresses the need for a digital platform where customers can browse available books, place orders, and track their order history — while store administrators can efficiently manage inventory, categories, and monitor sales.

1.2 Key Features

  • User Registration & Login — Secure JWT-based authentication
  • Book Catalog — Browse books with cover images, search by title/author, filter by category
  • Order Management — Place orders with quantity selection, automatic stock updates
  • Admin Dashboard — Statistics overview, CRUD operations for books/categories, order tracking
  • Role-Based Access Control — Separate permissions for USER and ADMIN roles
  • Responsive UI — Bootstrap-based design that works on all screen sizes
  • API Documentation — Auto-generated Swagger/OpenAPI docs

3.1 Backend Architecture (Layered Pattern)

  • Controller Layer — Handles HTTP requests, input validation, and response formatting
  • Service Layer — Contains business logic, transaction management
  • Repository Layer — JPA interfaces for database operations
  • Entity Layer — JPA entity classes mapped to database tables
  • Config Layer — Security, CORS, JWT, and Swagger configuration

6. Security Implementation

The application uses JWT (JSON Web Token) based authentication:

  1. User registers or logs in with email/password
  2. Server validates credentials and generates a JWT token containing user email and role
  3. Client stores the token in localStorage and sends it in the Authorization header
  4. JwtFilter intercepts requests, validates the token, and sets the security context
  5. Spring Security checks the user's role before granting access to protected endpoints

8. How to Run the Project

Prerequisites

  • Java 17 (JDK)
  • Node.js 18+ and npm
  • MySQL 8.x running on port 3306

Step 1: Setup Database

Create the MySQL database:

CREATE DATABASE bookstore_db;

Step 2: Start Backend

Navigate to bookstore-backend/ and run:

./mvnw spring-boot:run (Mac/Linux) or mvnw.cmd spring-boot:run (Windows)

Backend starts at http://localhost:8080

Step 3: Start Frontend

Navigate to bookstore-frontend/ and run:

npm install then npm run dev

Frontend starts at http://localhost:5173
